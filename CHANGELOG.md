# Changelog

All notable changes to the **Agent Flywheel Beads Viewer** static dashboard are documented here.

This project is a GitHub Pages-hosted, fully offline-capable single-page application generated by the [`bv` (beads_viewer)](https://github.com/Dicklesworthstone/beads_viewer) CLI tool. It provides interactive issue triage, dependency graph visualization, and project health analytics for beads-based projects.

## 2026-01-05 — Initial Deployment

### Full static site deployed via `bv --pages`
`[core]` `[deployment]` `[visualization]` `[search]` `[graph]` `[analytics]`

- **Commit**: [`a92d839`](https://github.com/Dicklesworthstone/beads_viewer_for_agentic_coding_flywheel_setup/commit/a92d839fcc43fc655815368d552a0f0b3bcfa605)
- **Date**: 2026-01-05 22:08:35 -0500
- **Author**: Jeff Emanuel
- **Live dashboard**: https://Dicklesworthstone.github.io/beads_viewer_for_agentic_coding_flywheel_setup/

This commit is the initial deployment of the entire static site, generated by `bv --pages`. It delivered the complete application in a single commit with no prior history. The capabilities shipped are described below.

#### Architecture

- **Fully self-contained SPA** — zero external network requests at runtime; all dependencies vendored locally
- **WASM-powered SQL engine** — client-side SQLite via `sql.js` (`vendor/sql-wasm.js` + `vendor/sql-wasm.wasm`) with FTS5 full-text search and materialized views
- **OPFS caching** — Origin Private File System caching for offline support and fast reloads
- **Cross-Origin Isolation** — service worker (`coi-serviceworker.js`) injects COOP/COEP headers to enable `SharedArrayBuffer` on GitHub Pages (where response headers cannot be configured)
- **Content Security Policy** — strict self-contained CSP meta tag (no external origins allowed)

#### Visualization: Force-Directed Dependency Graph

- **WASM graph engine** — `vendor/bv_graph.js` + `vendor/bv_graph_bg.wasm` for WASM-accelerated graph metrics (PageRank, betweenness centrality)
- **D3 + force-graph** — `vendor/d3.v7.min.js` and `vendor/force-graph.min.js` power the interactive force-directed layout
- **Dracula-inspired theme** — custom color palette for status (open/in_progress/blocked/closed), priority heat (P0-P4), and accent colors
- **Multiple view modes** — status coloring, priority heat, and graph-metric overlays
- **Rich interactions** — node hover tooltips, click-to-inspect, cycle detection highlighting, gold-glow for critical paths
- **Standalone demo** — `graph-demo.html` provides an isolated graph visualization page
- **Pre-computed layout** — `data/graph_layout.json` stores node positions and per-node metrics (PageRank, betweenness, etc.) for instant rendering

#### Search and Ranking: Hybrid Scorer

- **Multi-signal ranking** — `hybrid_scorer.js` implements graph-aware search ranking combining text relevance, PageRank, status, impact (blocker count), priority, and recency
- **Tunable weight presets** — five built-in presets: `default`, `bug-hunting`, `sprint-planning`, `impact-first`, `text-only`
- **WASM acceleration** — `wasm_loader.js` dynamically loads a WASM hybrid scorer (`bv_hybrid_scorer`) for datasets above 5,000 issues, with automatic JS fallback for smaller datasets or when WASM is unavailable
- **Test suite** — `hybrid_scorer.test.js` validates score computation against expected component weights
- **Normalization logic** — mirrors `pkg/search/hybrid_scorer_impl.go` from the Go codebase for cross-platform consistency

#### Analytics Dashboard

- **Chart.js-powered analytics** — `charts.js` provides four chart types:
  - Burndown/burnup progress chart
  - Label dependency heatmap (custom canvas renderer)
  - Priority distribution pie chart
  - Type breakdown bar chart
- **Project health data** — `data/project_health.json` with counts by status/type/priority, graph density, cycle detection, and 8-week velocity history

#### Triage Engine

- **AI-powered recommendations** — `data/triage.json` delivers scored issue recommendations with per-signal breakdowns (PageRank, betweenness, blocker ratio, staleness, priority boost, time-to-impact, urgency, risk)
- **Quick wins** — low-effort, high-value items surfaced automatically
- **Blocker analysis** — identifies blocked issues and suggests unblocking order
- **CLI integration** — triage data includes ready-to-run `bd` commands (`bd update`, `bd show`, `bd ready`, `bd blocked`, `bv --robot-triage`)

#### UI Framework

- **Alpine.js** — `vendor/alpine.min.js` + `vendor/alpine-collapse.min.js` for reactive UI with `x-cloak` flash prevention
- **Tailwind CSS** — `vendor/tailwindcss.js` (runtime JIT) with custom `beads` color scale and dark/light mode (`class`-based toggle with `localStorage` persistence, defaults to dark)
- **Vendored fonts** — Inter (variable, `vendor/inter-variable.woff2`) and JetBrains Mono (`vendor/jetbrains-mono-regular.woff2`) for offline typography
- **Markdown rendering** — `vendor/marked.min.js` with `vendor/dompurify.min.js` for safe HTML sanitization
- **Mermaid diagrams** — `vendor/mermaid.min.js` for rendering dependency diagrams in issue descriptions
- **Mobile-optimized** — PWA-capable meta tags, viewport-fit cover, theme-color for light/dark, touch-friendly layout

#### Data Layer

- **SQLite database** — `beads.sqlite3` (39 KB) with issue data, queryable client-side
- **Chunk configuration** — `beads.sqlite3.config.json` defines chunking parameters (this deployment: single file, no chunking)
- **JSON data files**:
  - `data/meta.json` — version, generation timestamp, issue/dependency counts
  - `data/graph_layout.json` — pre-computed node positions and graph metrics
  - `data/project_health.json` — status/type/priority breakdowns, graph stats, velocity
  - `data/triage.json` — scored recommendations, quick wins, blocker analysis

#### Diagnostics and Error Handling

- **Structured error display** — `showError()` with title, message, technical details, action buttons, and dismissibility
- **Diagnostics state** — tracks WASM load status (sql.js, bv_graph, hybrid scorer), database source, size, query count/errors, and load time
- **Toast notifications** — non-blocking status messages via custom events
- **Safe query wrapper** — `safeQuery()` catches SQL errors, tracks query statistics, returns structured results with fallbacks
