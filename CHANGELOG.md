# Changelog

All notable changes to the **Agent Flywheel Beads Viewer** static dashboard are documented here.

This project is a GitHub Pages-hosted, fully offline-capable single-page application generated by the [`bv` (beads_viewer)](https://github.com/Dicklesworthstone/beads_viewer) CLI tool. It provides interactive issue triage, dependency graph visualization, and project health analytics for beads-based projects.

> **Format**: This changelog is organized chronologically by commit date (newest first). There are no tagged releases or GitHub Releases in this repository; every entry references a commit. Links point to the canonical GitHub commit URL.
>
> **For agents**: Each entry includes the commit hash, date, author, and a capability-oriented summary. Search for `[capability]` tags to find entries by domain.

## 2026-01-05 — Initial Deployment

### Full static site deployed via `bv --pages`
`[core]` `[deployment]` `[visualization]` `[search]` `[graph]` `[analytics]`

- **Commit**: [`a92d839`](https://github.com/Dicklesworthstone/beads_viewer_for_agentic_coding_flywheel_setup/commit/a92d839fcc43fc655815368d552a0f0b3bcfa605)
- **Date**: 2026-01-05 22:08:35 -0500
- **Author**: Jeff Emanuel
- **Live dashboard**: https://Dicklesworthstone.github.io/beads_viewer_for_agentic_coding_flywheel_setup/

This commit is the initial deployment of the entire static site, generated by `bv --pages`. It delivered the complete application in a single commit (32 files, ~19,800 lines) with no prior history. The capabilities shipped are organized below by domain.

#### Self-Contained Offline Architecture
`[architecture]` `[offline]` `[security]`

- **Fully self-contained SPA** with zero external network requests at runtime; all 14 vendor dependencies bundled locally (Alpine.js, Chart.js, D3 v7, force-graph, sql.js, Mermaid, Marked, DOMPurify, Tailwind CSS, plus fonts and WASM binaries)
- **WASM-powered SQL engine** via `sql.js` (`vendor/sql-wasm.js` + `vendor/sql-wasm.wasm`) enabling client-side SQLite queries with FTS5 full-text search and materialized views
- **OPFS caching** using Origin Private File System for offline support and fast subsequent loads; database cached after first network fetch
- **Cross-Origin Isolation** service worker (`coi-serviceworker.js`) injects COOP/COEP headers to enable `SharedArrayBuffer` on GitHub Pages, using the `credentialless` policy to allow cross-origin resources without credentials
- **Content Security Policy** via strict `<meta>` tag restricting all resource origins to `'self'`
- **Database chunking support** for large databases (`beads.sqlite3.config.json` configures chunk reassembly); this deployment uses a single 39 KB file

#### Force-Directed Dependency Graph
`[graph]` `[visualization]` `[wasm]`

- **WASM graph engine** (`vendor/bv_graph.js` + `vendor/bv_graph_bg.wasm`) computing eight graph metrics in real-time:
  - PageRank (importance/influence)
  - Betweenness centrality (bottleneck detection, with approximate mode for graphs > 500 nodes)
  - Eigenvector centrality (influence propagation)
  - K-core decomposition (cohesion/clique identification)
  - HITS hub and authority scores (hyperlink-induced topic search)
  - Critical path heights (longest-path depth from roots)
  - Slack values (schedule flexibility; zero slack = on critical path)
  - Articulation points (cut vertices whose removal disconnects the graph)
  - Cycle enumeration (up to 100 cycles detected)
- **D3 v7 + force-graph** rendering with canvas-based custom node/link drawing
- **Five view modes** switchable via keyboard (1-5):
  1. Force-directed (standard physics-based layout)
  2. Hierarchy (top-down DAG tree)
  3. Radial (circular layout from center/selection)
  4. Cluster (grouped by status)
  5. Label Galaxy (clustered by label with convex hull overlays and label legends)
- **Six layout presets** with tunable simulation parameters: Force-Directed, Compact DAG, Spread, Orthogonal, Radial, Cluster
- **What-if simulation** (Shift+click or `w` key): animated cascade visualization showing which issues would be unblocked if a selected issue were closed, with green/cyan glow states and summary popup
- **Critical path animation** (`c` key): animated traversal highlighting the longest dependency chain with red glow, step-by-step playback, and summary display
- **Cycle navigator** (`y` key): step through detected dependency cycles one at a time with `[`/`]` navigation and zoom-to-cycle
- **Heatmap mode** (`h` key): color nodes by PageRank, betweenness, critical path depth, or in-degree; dynamically resizes nodes proportional to the selected metric
- **Golden glow subgraph** highlighting: hovering a node illuminates its 2-hop neighborhood with gold glow effects on connected nodes and links
- **Rich node rendering**: priority flame indicators for P0/P1 issues, cycle warning icons, zoom-adaptive labels (title at high zoom, ID at low zoom), type-based emoji icons
- **Interactive tooltips** with issue ID, title, status, priority, PageRank, betweenness, and labels
- **Pre-computed layout** (`data/graph_layout.json`) stores node positions and per-node metrics for instant initial rendering
- **Standalone demo** page (`graph-demo.html`) with independent Dracula-themed graph visualization

#### Time-Travel Animation
`[graph]` `[history]`

- **Commit history playback** (`t` key to toggle, Space to play/pause): animates the graph through historical snapshots from `data/history.json` (optional), showing issues appearing/disappearing over time
- Transport controls with playback speed slider, timeline scrubber, and commit info display
- Node states tracked per-commit with opacity and visibility animations

#### Hybrid Search and Ranking
`[search]` `[ranking]` `[wasm]`

- **Multi-signal hybrid scorer** (`hybrid_scorer.js`) combining six ranking dimensions: text relevance, PageRank, status weight, impact (blocker count), priority, and recency (exponential decay over 30 days)
- **Five tunable weight presets**: `default`, `bug-hunting`, `sprint-planning`, `impact-first`, `text-only`
- **WASM acceleration** (`wasm_loader.js`): dynamically loads a WASM hybrid scorer for datasets above 5,000 issues, with automatic JavaScript fallback for smaller datasets
- **Adaptive query weights**: short queries boost PageRank/status; issue-ID-like queries get exact-match promotion
- **FTS5 full-text search** with LIKE-based fallback; token-counting heuristics adjust search strategy
- **Test suite** (`hybrid_scorer.test.js`) validates score computation against expected component weights
- **Cross-platform consistency** with normalization logic mirroring `pkg/search/hybrid_scorer_impl.go` from the Go codebase

#### Analytics Dashboard
`[analytics]` `[charts]`

- **Chart.js-powered analytics** (`charts.js`) with Dracula-themed styling:
  - Burndown/burnup progress chart tracking open vs. closed issues over time
  - Priority distribution pie chart (P0-P4 with color-coded slices)
  - Type breakdown bar chart (bug/feature/task/epic/chore)
- **Project health data** (`data/project_health.json`) with counts by status/type/priority, graph density metric, cycle detection flag, and 8-week velocity history

#### Triage Engine and Insights
`[triage]` `[insights]` `[analytics]`

- **AI-powered recommendations** (`data/triage.json`) with per-issue scored breakdowns across eight signals: PageRank, betweenness, blocker ratio, staleness, priority boost, time-to-impact, urgency, and risk
- **Insights view** (`#/insights`) surfacing:
  - Top issues by PageRank, betweenness, critical path depth, HITS hub/authority, K-core, and triage score
  - Articulation points (graph cut vertices)
  - Critical path slack analysis (zero-slack issues)
  - Cycle detection summaries
  - Top what-if impact issues (which closures would unblock the most)
- **Quick wins** and **blockers-to-close** suggestions
- **Score breakdown visualization** with per-signal percentage bars and normalized values
- **Raw triage JSON viewer** modal for debugging

#### SPA Navigation and URL State
`[navigation]` `[routing]` `[sharing]`

- **Hash-based SPA router** with five routes: Dashboard (`/`), Issues list (`/issues`), Issue detail (`/issue/:id`), Insights (`/insights`), Graph (`/graph`)
- **Shareable filtered views**: filter state (status, type, priority, labels, assignee, blocked/blocking flags, search query, sort order) serialized to URL query parameters
- **Browser history integration**: back/forward navigation via `hashchange` events; `history.replaceState` for filter updates without creating history entries
- **Deep linking** to individual issues via `#/issue/{id}`

#### Keyboard-Driven Navigation
`[accessibility]` `[keyboard]`

- **Global shortcuts**: `/` to focus search, `?` for keyboard help modal, `d` for diagnostics panel
- **Graph shortcuts**: `1`-`5` for view modes, `w` for what-if, `c` for critical path, `y` for cycle navigator, `h` for heatmap, `[`/`]` for cycle stepping, `t` for time-travel, Space for play/pause, `r` to reset view, `Escape` to dismiss
- **Issue navigation**: `j`/`k` for vim-style list traversal, `o` to open detail, `h`/`l` to traverse blocker/dependent chains
- **Arrow key navigation**: Left/Right to navigate by priority order (PageRank-ranked within each priority level), Up/Down to navigate between K-core cliques
- **Ctrl/Cmd+F** intercepted to focus in-app search

#### UI Framework and Theming
`[ui]` `[theming]` `[mobile]`

- **Alpine.js** (`vendor/alpine.min.js` + `vendor/alpine-collapse.min.js`) for reactive data binding with `x-cloak` flash prevention
- **Tailwind CSS** runtime JIT (`vendor/tailwindcss.js`) with custom `beads` color scale (50-900)
- **Dark/light mode** via class-based toggle with `localStorage` persistence, defaulting to dark; early `<script>` prevents flash-of-unstyled-content; respects `prefers-color-scheme` system preference when no stored choice exists
- **Dracula-inspired palette** for graph visualization: status colors (green/orange/red/muted), priority heat (red to muted), and accent colors (purple, pink, cyan, green, orange, red, yellow, gold)
- **Vendored fonts**: Inter variable (`vendor/inter-variable.woff2`) and JetBrains Mono (`vendor/jetbrains-mono-regular.woff2`) for fully offline typography
- **Markdown rendering** via `vendor/marked.min.js` with HTML sanitization through `vendor/dompurify.min.js`; supports inline and block rendering modes plus plain-text stripping for excerpts
- **Mermaid diagrams** (`vendor/mermaid.min.js`) for rendering dependency flowcharts in issue descriptions
- **Syntax-highlighted JSON** display for triage data and raw issue details
- **Mobile-optimized**: PWA-capable meta tags (`mobile-web-app-capable`, `apple-mobile-web-app-capable`), `viewport-fit: cover`, dual `theme-color` for light/dark, hamburger menu, collapsible filter panels, body scroll lock for modals (iOS scroll bleed fix), reduced-motion-aware scroll behavior
- **Toast notification system** via custom events with auto-dismiss (5 second timeout)

#### Error Handling and Diagnostics
`[diagnostics]` `[error-handling]`

- **Structured error display** with title, message, technical details, action buttons, and dismissibility control
- **Diagnostics panel** (`d` key) tracking: WASM load status (sql.js, bv_graph, hybrid scorer + failure reason), OPFS availability, database source (network/cache/chunks) and size, issue count, total load time, query count and error count
- **Safe query wrapper** (`safeQuery()`) catches SQL errors, increments query statistics, and returns structured results with fallbacks
- **WASM feature detection**: probes for basic WASM support, SIMD, and SharedArrayBuffer (threads); degrades gracefully to pre-computed data in fallback mode
- **WASM memory management**: cleanup on page unload and visibility change (handles mobile browser tab eviction)

#### Data Layer
`[data]` `[database]`

- **SQLite database** (`beads.sqlite3`, 39 KB) with issue data queryable entirely client-side
- **Chunk configuration** (`beads.sqlite3.config.json`) defines chunking parameters for large database support
- **JSON data files**:
  - `data/meta.json` — version (1.0.0), generation timestamp, issue/dependency counts
  - `data/graph_layout.json` — pre-computed node positions and per-node graph metrics
  - `data/project_health.json` — status/type/priority breakdowns, graph density, cycle detection, 8-week velocity
  - `data/triage.json` — scored recommendations with per-signal breakdowns, quick wins, blocker analysis
